# EXPLANATION 35 1

**Question 1**

**What is true about functions registered in Unity Catalog?**

**Correct Answer:** **C. They are securable objects that can be shared, managed, and granted permissions.**

**Explanation**

Functions registered in Unity Catalog are **first-class securable objects**. Like tables and views, administrators can grant permissions on them, manage ownership, and share them across users and workspaces. Unity Catalog supports SQL User-Defined Functions (UDFs), Python UDFs, stored procedures, and registered ML models as function objects.

**Why the Correct Answer is Correct**

Unity Catalog functions:

- Are permanently registered.

- Can be governed using Unity Catalog permissions.

- Can be shared across users.

- Can be reused by multiple notebooks and SQL queries.

**Why the Other Answers Are Wrong**

- They incorrectly imply functions are temporary.

- They suggest functions are limited to a single notebook or Spark session.

- They ignore Unity Catalog governance capabilities.
