# EXPLANATION 44 2

**Question 2**

**Which of the following is a common use case for Databricks SQL Alerts?**

**Correct Answer:**\
**B. Monitoring pending order counts, data quality issues, and business KPIs**

**Why B is Correct**

SQL Alerts are commonly used to monitor:

- Pending orders

- Failed transactions

- Revenue thresholds

- Missing data

- Data quality metrics

- Inventory levels

- SLA violations

These are ideal because they can all be expressed as SQL queries with threshold conditions.

**Why the other choices are wrong**

Alerts are not intended for:

- Writing ETL pipelines

- Training ML models

- Managing clusters

- Creating Unity Catalog objects

**Memory Tip**

**Anything measurable with SQL can be monitored.**
