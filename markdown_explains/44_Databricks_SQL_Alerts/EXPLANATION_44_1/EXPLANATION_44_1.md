# EXPLANATION 44 1

**Question 1**

**What is the primary purpose of Databricks SQL Alerts?**

**Correct Answer:**\
**B. To continuously monitor SQL query results and send notifications when specified conditions are met**

**Why B is Correct**

A Databricks SQL Alert automatically:

- Runs a SQL query on a schedule.

- Evaluates the query results against a condition (threshold).

- Sends notifications if the condition is met.

Examples include:

- Data quality monitoring

- Business KPI monitoring

- Cost monitoring

- Operational monitoring

This allows organizations to proactively detect issues without manually checking dashboards.

**Why the other choices are wrong**

SQL Alerts do **not**:

- replace SQL Warehouses

- create dashboards

- modify data automatically

- ingest streaming data

**Memory Tip**

**SQL Alert = Query + Condition + Notification**
