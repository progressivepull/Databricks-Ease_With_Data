# EXPLANATION 40 1

**Question 1**

**What is the primary purpose of Databricks Serverless Compute?**

**Correct Answer:**\
**B. To automatically manage compute infrastructure, including cluster creation, scaling, and termination**

**Why B is Correct**

The biggest benefit of **Serverless Compute** is that **Databricks manages all infrastructure** for you.

Instead of users:

- Creating clusters

- Selecting VM sizes

- Configuring autoscaling

- Terminating clusters

- Installing runtime updates

Databricks automatically handles these tasks.

This allows engineers to focus entirely on writing code instead of managing infrastructure.

**Key idea**

Serverless = **No infrastructure management**

------------------------------------------------------------------------

**Why the others are wrong**

**A. Automatically creates Unity Catalog**

No.

Unity Catalog is a governance system.

Serverless Compute only executes workloads.

------------------------------------------------------------------------

**C. Stores Delta tables**

Wrong.

Delta tables are stored in cloud storage.

Serverless only processes them.

------------------------------------------------------------------------

**D. Creates dashboards**

Dashboards are created in DBSQL.

Serverless provides compute.

------------------------------------------------------------------------

**E. Replaces Spark**

Spark is still the execution engine.

------------------------------------------------------------------------

**Memory Tip**

**Classic Compute**

You manage the cluster.

**Serverless**

Databricks manages the cluster.
