# EXPLANATION 48 8

**Question 8**

**After modifying a notebook, what indication does Databricks display before the changes are committed?**

**Correct Answer:**\
**B. Modified (M)**

**Why B is Correct**

Git tracks modified files.

Example:

Notebook.py M

README.md M

The **M** indicates the file has changed locally but has not yet been committed.

After committing:

Notebook.py

README.md

The indicator disappears because the changes are now part of the Git history.

**Why the other choices are wrong**

Other indicators such as staged or pushed represent different Git states.

**Memory Tip**

**M = Modified**
