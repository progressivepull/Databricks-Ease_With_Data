# EXPLANATION 48 9

**Question 9**

**Which sequence correctly describes the Git workflow demonstrated in the lesson?**

**Correct Answer:**\
**B. Create Notebook → Edit Notebook → Commit → Push**

**Why B is Correct**

The standard Git workflow is:

Create

↓

Edit

↓

Commit

↓

Push

- **Create** a notebook or file.

- **Edit** it locally.

- **Commit** changes to the local Git history.

- **Push** commits to the remote repository.

This sequence ensures work is versioned before it is shared with collaborators.

**Why the other choices are wrong**

Pushing before committing is not possible because Git pushes commits, not uncommitted changes.

**Memory Tip**

**Edit → Commit → Push**
