# EXPLANATION 48 1

**Question 1**

**What is the primary purpose of Databricks Git Folders?**

**Correct Answer:**\
**B. To integrate Git repositories with a Databricks workspace for version control and collaboration**

**Why B is Correct**

Databricks Git Folders allow developers to connect a remote Git repository (GitHub, Azure DevOps, GitLab, Bitbucket, etc.) directly to their Databricks workspace.

Benefits include:

- Version control

- Team collaboration

- Branch management

- Pull Requests

- CI/CD integration

- Code history

Instead of copying notebooks manually, Git keeps the workspace synchronized with a source repository. Databricks Git Folders support common Git operations such as clone, pull, push, commit, branch management, merge, and rebase.

**Why the other choices are wrong**

Git Folders do **not**:

- replace Unity Catalog

- create SQL Warehouses

- manage Spark clusters

- store Delta tables
