# EXPLANATION 48 7

**Question 7**

**Why does the lesson recommend saving Databricks notebooks in Source format instead of the default .ipynb format?**

**Correct Answer:**\
**B. Source format stores notebooks as .py files, making Git diffs, code reviews, and version control easier.**

**Why B is Correct**

Notebook source format stores notebooks as plain text files such as:

- .py

- .sql

- .scala

- .r

Advantages:

- Smaller Git commits

- Easier code reviews

- Cleaner diffs

- Better merge conflict resolution

Notebook JSON (.ipynb) files are much harder to compare in Git.

**Why the other choices are wrong**

Source format is recommended for maintainability and collaboration, not because it executes faster.

**Memory Tip**

**Git likes text files.**
