# EXPLANATION 05 6

**Question 6**

**What is a Service Principal in Databricks primarily used for?**

**Correct Answer:**\
✅ **Acting as a non-human identity for automation and production workloads**

**Explanation**

A **Service Principal** is an identity for software rather than a person.

Typical uses:

- CI/CD pipelines

- Automated Jobs

- Azure DevOps

- GitHub Actions

- Scheduled Workflows

- Production applications

Instead of storing someone's username and password, automation authenticates using a Service Principal.

Benefits:

- Better security

- Easier credential rotation

- No dependence on an individual employee
