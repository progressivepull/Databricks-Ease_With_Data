# EXPLANATION 45 1

**Question 1**

**What is the primary purpose of a Metric View in Databricks Unity Catalog?**

**Correct Answer:**\
**B. To provide a centralized, reusable, governed semantic layer for business metrics and dimensions**

**Why B is Correct**

A **Metric View** creates a **semantic layer** inside Unity Catalog.

Instead of every dashboard developer writing their own SQL calculations, a Metric View defines business metrics **once** so everyone uses the same trusted definitions.

Typical examples include:

- Revenue

- Total Orders

- Profit Margin

- Customer Count

- Conversion Rate

These definitions become reusable across:

- AI/BI Dashboards

- Genie

- SQL

- Notebooks

This ensures consistent KPI calculations across the organization.

**Why the other choices are wrong**

Metric Views are **not**:

- storage tables

- ETL pipelines

- SQL Warehouses

- Machine Learning models

**Memory Tip**

**Metric View = Semantic Layer**
