# EXPLANATION 01 2

A **key characteristic of a data lake** is that it stores **large volumes of raw data in its native format** (structured, semi-structured, and unstructured) without requiring a predefined schema.

**Explanation**

Unlike a traditional data warehouse, where data must be cleaned and organized before storage (**schema-on-write**), a data lake uses **schema-on-read**. This means:

- Data is stored as-is.

- The structure is applied only when the data is analyzed.

- It can hold diverse data types such as databases, logs, documents, images, videos, and IoT sensor data.

**Other important characteristics**

- **Scalability:** Can store massive amounts of data cost-effectively.

- **Flexibility:** Supports multiple data formats.

- **Advanced analytics support:** Useful for machine learning, AI, and big data processing.

- **Centralized storage:** Combines data from many sources in one repository.

**Example:** A company may store customer transactions, website clickstreams, social media posts, and product images together in a data lake, then analyze them later for business insights.

**In one sentence:** A data lake's defining feature is its ability to store vast amounts of **raw, diverse data without requiring it to be structured beforehand**.
