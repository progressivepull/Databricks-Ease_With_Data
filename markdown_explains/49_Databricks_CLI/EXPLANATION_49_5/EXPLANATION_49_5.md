# EXPLANATION 49 5

**Question 5**

**Which authentication method is best suited for local development and interactive use?**

**Correct Answer:**\
**B. User-to-Machine (U2M) using a user account**

**Why B is Correct**

**User-to-Machine (U2M)** authentication is designed for developers working interactively.

Typical workflow:

1.  Sign in as yourself.

2.  Authenticate through Databricks.

3.  Use the CLI with your own user identity.

Benefits include:

- Easy setup

- Interactive sign-in

- Uses your personal permissions

([docs.databricks.com](https://docs.databricks.com/aws/en/dev-tools/auth/))

**Why the other choices are wrong**

Machine-to-Machine (M2M) authentication is intended for automation, not day-to-day interactive development.
