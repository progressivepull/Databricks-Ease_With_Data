# EXPLANATION 49 1

**Question 1**

**What is the primary purpose of the Databricks CLI (Command Line Interface)?**

**Correct Answer:**\
**B. To manage Databricks accounts and workspaces using terminal commands**

**Why B is Correct**

The Databricks CLI allows administrators and developers to manage Databricks resources directly from a terminal or command prompt.

Common tasks include:

- Managing workspaces

- Managing Unity Catalog

- Creating jobs

- Deploying Asset Bundles

- Managing files

- Running automation scripts

- Managing account-level resources (where supported)

The CLI is especially useful for automation and CI/CD workflows because it eliminates the need to perform repetitive tasks through the web UI. ([docs.databricks.com](https://docs.databricks.com/aws/en/dev-tools/cli/))

**Why the other choices are wrong**

The CLI does **not**:

- replace SQL Warehouses

- replace Unity Catalog

- train machine learning models

- store Delta tables
