# ANSWER 01 6

**What is the primary purpose of Delta Live Tables (DLT) in Databricks?**

A. To create interactive business intelligence dashboards\
**B. To automate and manage reliable data pipelines with built-in quality checks and monitoring**\
C. To replace Unity Catalog for data governance and access control\
D. To optimize cluster hardware performance for machine learning workloads\
E. To store only unstructured streaming video data

**Correct Answer:** B. To automate and manage reliable data pipelines with built-in quality checks and monitoring

------------------------------------------------------------------------

Let’s break down each option so you clearly understand **why it’s correct or incorrect**.

**✅ Correct Answer: B**

**“To automate and manage reliable data pipelines with built-in quality checks and monitoring”**

This is exactly what **Delta Live Tables (DLT)** is designed for:

- Automates ETL/ELT pipelines

- Simplifies data engineering workflows

- Includes built-in:

  - Data quality rules

  - Pipeline monitoring

  - Error handling

  - Dependency management

👉 DLT helps create **reliable, maintainable, and scalable data pipelines**

------------------------------------------------------------------------

**❌ A. To create interactive business intelligence dashboards**

**Why wrong:**

- Dashboards are created using:

  - Databricks SQL

  - Power BI

  - Tableau

- DLT focuses on **data pipelines**, not visualization

👉 Pipeline tool ≠ dashboard tool

------------------------------------------------------------------------

**❌ C. To replace Unity Catalog for data governance and access control**

**Why wrong:**

- Unity Catalog handles:

  - Governance

  - Permissions

  - Data lineage

- DLT handles:

  - Data processing pipelines

👉 They work together, but serve different purposes

------------------------------------------------------------------------

**❌ D. To optimize cluster hardware performance for machine learning workloads**

**Why wrong:**

- Cluster optimization is managed through:

  - Spark configurations

  - Compute settings

- DLT is not a hardware optimization tool

👉 DLT manages workflows, not infrastructure tuning

------------------------------------------------------------------------

**❌ E. To store only unstructured streaming video data**

**Why wrong:**

- DLT is not a storage system

- It processes many types of data:

  - Structured

  - Semi-structured

  - Streaming data

👉 Not limited to video or unstructured data

------------------------------------------------------------------------

**🔑 Simple Summary**

- **Delta Live Tables = Automated Data Pipelines**

- Main strengths:

  - Reliability

  - Automation

  - Monitoring

  - Data quality enforcement

👉 That’s why **B is correct**
