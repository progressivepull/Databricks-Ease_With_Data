# ANSWER 06 9

**Question 9**

**Which Databricks notebook feature allows developers to restore previous versions of a notebook?**

**A. Spark UI** ❌ Incorrect

- Spark UI shows execution details and job performance.

- It does not store notebook versions.

**B. Variable Explorer** ❌ Incorrect

- Variable Explorer displays variables during execution.

- It does not provide version control.

**C. Version History** ✅ **Correct**

- Version History automatically saves notebook revisions.

- Developers can:

  - View previous versions

  - Compare changes

  - Restore earlier versions if needed

**D. Catalog Explorer** ❌ Incorrect

- Catalog Explorer manages data assets in Unity Catalog.

- It is unrelated to notebook versioning.

**E. Cluster Metrics** ❌ Incorrect

- Cluster Metrics monitor compute performance.

- They do not track notebook edits.

**Key Concept**

Version History acts like an **undo history across sessions**, allowing you to recover earlier notebook versions.
