# ANSWER 41 8

**Question 8**

**What information can users analyze in the Query Profile after executing a SQL query?**

**A. Only the SQL query text** ❌ Incorrect

- The Query Profile contains much more.

**B. Query execution plan, execution time, memory usage, rows processed, and data scanned** ✅ Correct

- Query Profile helps diagnose performance by showing:

  - Execution plan

  - Operator breakdown

  - Execution time

  - Memory usage

  - Data scanned

  - Rows processed

This helps identify slow operations.

**C. Only cluster startup time** ❌ Incorrect

- Startup time is only a small aspect and not the focus.

**D. Only Delta transaction history** ❌ Incorrect

- Delta transaction history is separate.

**E. Only user permissions for the query** ❌ Incorrect

- Permissions aren't the purpose of Query Profile.

**Key Concept**

**Query Profile = SQL Performance Troubleshooting Tool**
