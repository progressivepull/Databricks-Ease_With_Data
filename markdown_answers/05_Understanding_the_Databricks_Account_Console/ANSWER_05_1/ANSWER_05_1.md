# ANSWER 05 1

**Question 1**

**What is the primary purpose of the Databricks Account Console?**

**A. To develop Spark notebooks only** ❌ Incorrect

- Spark notebooks are created and edited inside a **Databricks workspace**, not the Account Console.

- The Account Console focuses on administration rather than development.

**B. To monitor cluster CPU utilization only** ❌ Incorrect

- Cluster monitoring occurs within the workspace's Compute section and monitoring tools.

- The Account Console manages far more than cluster performance.

**C. To centrally manage Databricks accounts, workspaces, users, and settings** ✅ **Correct**

- This is the primary purpose of the Account Console.

- It provides centralized administration across an entire Databricks account.

- Administrators can:

  - Create and manage workspaces

  - Manage users and groups

  - Configure Service Principals

  - Configure networking

  - Enable account-wide features

  - Manage Unity Catalog and metastores

**D. To replace Unity Catalog** ❌ Incorrect

- Unity Catalog is a governance service.

- The Account Console manages and configures Unity Catalog but does not replace it.

**E. To create Delta tables only** ❌ Incorrect

- Delta tables are created inside workspaces using SQL, notebooks, or pipelines.

- The Account Console has nothing to do with creating tables.

**Key Concept**

Think of the Account Console as the **headquarters** for your Databricks environment. It manages the entire organization, while each workspace is where users actually build and run workloads.
