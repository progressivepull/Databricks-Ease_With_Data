Question 1
What is the primary purpose of Object-Level Security in Unity Catalog?
A. To automatically optimize Delta tables for performance
B. To encrypt all data stored in Unity Catalog
C. To control who can view, access, modify, and manage individual Unity Catalog objects
D. To create catalogs and schemas automatically
E. To manage Spark cluster configurations
Correct Answer: C. To control who can view, access, modify, and manage individual Unity Catalog objects

Question 2
Which of the following correctly represents the Unity Catalog object hierarchy?
A. Schema → Catalog → Metastore → Table
B. Metastore → Catalog → Schema → Tables, Views, Volumes, Functions, and Models
C. Catalog → Metastore → Schema → Views
D. Table → Schema → Catalog → Metastore
E. Volume → Table → Schema → Catalog
Correct Answer: B. Metastore → Catalog → Schema → Tables, Views, Volumes, Functions, and Models

Question 3
What happens when a privilege is granted on a parent object, such as a Catalog?
A. Permissions apply only to the Catalog itself.
B. Child objects automatically inherit the permission.
C. All existing permissions on child objects are removed.
D. Only Metastore Administrators receive the permission.
E. The permission applies only after a manual refresh.
Correct Answer: B. Child objects automatically inherit the permission.

Which approach is recommended for repeatable, automated permission management in Unity Catalog?

B. Using only the Catalog Explorer UI
