Question 1
What is the primary purpose of a Streaming Table in Databricks SQL (DBSQL)?
A. To permanently archive Delta transaction logs
B. To continuously ingest incremental data from a streaming source
C. To replace SQL Warehouses for query execution
D. To store only precomputed aggregations for dashboards
E. To automatically create Unity Catalog catalogs
Correct Answer: B. To continuously ingest incremental data from a streaming source

Question 2
Although Streaming Tables and Materialized Views are created using SQL in DBSQL, what technology manages refreshes and incremental processing behind the scenes?
A. Apache Airflow
B. Databricks Workflows
C. Delta Live Tables (DLT)
D. Apache Hive

Question 6
What is the effect of setting includeExistingFiles = false when creating a Streaming Table?
A. All existing files are reprocessed during every refresh.
B. Previously processed files are ignored, and only newly arriving files are processed.
C. Streaming Tables cannot be refreshed incrementally.
D. The Streaming Table performs a full refresh every hour.

B. It deletes all cached results.
C. It performs a No Operation (NO OP), skipping unnecessary processing.

Question 10
According to best practices, what is the correct refresh order when using both Streaming Tables and Materialized Views?
A. Refresh the Materialized View first, then the Streaming Table.
B. Refresh both simultaneously.
C. Refresh the Streaming Table first, then refresh the Materialized View.
D. Only refresh the Materialized View because the Streaming Table refreshes automatically.
